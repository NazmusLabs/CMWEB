One of the great things about CSS is the ability to create layouts for a web page without needing to rely on tables. During the early days of the web, webmasters (ðŸ˜‰) made extensive use of tables to design their web pages. Tables offered rows, columns, and cells, providing theneeded divisions that were needed used to organize blocks of texts, site navigation hyperlinks, and images.

But tables were designed primarily to represent data in an HTML document, and they are still one of the best ways to use them today. On the other hand, tables aren't the best nor most flexible for organizing complex HTML web pages.

I will use this lab to demonstrate (insha'Allah) how we can use the div element to seperate blocks of text in an HTML document and then use CSS to position and resize these blocks to begain laying out our web page. CSS allows us to do everything that we could do using tables and much more.

Creating a Navigation Bar

Fundamentally, the typical navigation bar (menu bar) we see on many websites is simply an unordered list as far as HTML is concerned. This applies to both horizontal and vertical navigation menus. It is CSS that specifies how to display that unordered list. Let's take a look at this in action.

Divs and Floats

Another way tables allowed webmasters to organize their webpage content was by dividing sections of text or images into separate blocks. For example, one might use a table to simulate a multi-column paragraph. Today, we use the "div" HTML element to separate blocks of HTML content.

By default, blocked off sections of content in an HTML document are represented sequentially from top to bottom. So, the one "div" element, for example, containing some text will display underneath another div containing an image that directly precedes it. We can use the "float" property in CSS to change how and where the individual blocks of HTML content display.

CSS Properties

Some CSS properties for webpage layouts:

  • Property: "display"; Possible Values: "block, contents, flex, flow-root, grid"
  • Property: "padding"; Possible Values: Up to four attributes, going clickwise; can be in px, em, or another unit. "padding-top (value) padding-right (value) padding-bottom (value) padding-left (value)"
  • Property: "margin"; Possible Values: Same format as padding. "margin-top (value) margin-right (value) margin-bottom (value) margin-left (value)"
  • Property: "float"; Possible Values: "left, right, inline-start, inline-end, none"
  • Property: "box-sizing"; Possible Values: "content-box, border-box"

The Importance of Responsive Design

Having a responsive design is essential on many levels. The number one reason to use responsive design is due to the multitude of devices we use to browse the web, which can range from a four-inch iPhone SE to a 32-inch 4k desktop monitor. (Technically, yes, we do have 80 inch TVs that can browse the web, but they typically do not have the resolution a high end, a 32-inch monitor would have). A responsive website allows the content to flow fluidly such that they fit the screen's width without needing to resort to horizontal scrolling.

P.S. This website has some responsive elements. If you zoom in on your browser far enough, you may notice that the sidebar navigation menu reflows to the center of the page.

More on Responsive Design

When smartphones were starting to become mainstream computing devices, many websites would have a separate mobile version of their website independent of their desktop counterparts. These mobile sites were designed to work well with smaller screens but often lacked the features of the desktop versions. Today, however, most webpages are responsive, which helps ensure that both the mobile and desktop users can experience the same website and features while still enjoying a layout optimized for their screen.

Accessibility for the Visually Impaired

There is, however, a second but less discussed reason to have responsive designs, one that affects me and many others directly. I am visually impaired, which means that for me to be able to see the text on a web page clearly at 100% zoom on a 1080p 27-inch screen, I need to bring my eyes close to one inch to the monitor. Before responsive designs became common, zooming in on the browser meant I had to scroll back and forth using the horizontal scrollbar, making it difficult to view websites at zoom levels of 150% to 300%, which typically is what I require to read comfortably. With designs that are responsive, however, I can zoom in on the browser to more than 300%. With content reflowing perfectly, I can enjoy a "mobile" view even on larger PC displays, enabling me to read the content comfortably without needing to pan back and forth horizontally or bring my eyes right in front of the screen. So by making your website responsive, you are ensuring that it is more accessible for visually impaired people like me!
