Cascading Style Sheets, or CSS, allows one to specify how the different elements in a markup (in our case HTML) should render. There are three ways to integrate CSS in an HTML document. The simplest way to do this so is by using what is known as an "inline styling"; this involves inserting CSS directly in the element one wishes to style. For a more global approach, however, we can use either an embedded style or an external stylesheet.

Embedded vs External

Both embedded and external stylesheets enable us to globally apply the same style to multiple occurrences of an element in the HTML document. The former involves including CSS in the header of an HTML document to which the styling is applied. The latter is when the HTML document in question references a separate file containing the CSS. These external files have a ".css" extension. This lab demonstrates all of these methods.

In the header of this HTML page (Ctrl + U to view source), we can see that an external stylesheet, lab3.css, is imported, below which we have additional CSS. These extra lines of CSS are embedded styles, and it overrides any conflicts found in the external file because the import line comes before the embedded CSS. Additionally, some of the paragraphs below have inline stiling that changes their font size and color.

In our case, for instance, the external CSS declares that the h1 and h2 elements are left-aligned and colored consistently with the rest of the text. However, the CSS embedded specify the same h1 and h2 tags to be center-aligned and colored in a shade of purple.

Web Resources

Traversy Media is a fantastic YouTube channel, especially for those who are new to web development like I was when I started this program here at Illinois Central College. This channel covers a wide breadth of web-developer related topics, including HTML, CSS, JS, and PHP. Their CSS playlist is an excellent resource for learning to do creative things with CSS.

The CSS Reference page from W3School is the place to go to look up all of the properties and their values that are part of the latest CSS specification. It is a place where I go for documentation on anything unfamiliar I come across as I design my pages.

CSS Tricks is a website I have used several times in my CMWEB 110 course and one that I continue to use in the CMWEB 120 course. This resource provides a wide variety of tips and guidelines for writing good CSS. It also contains articles that explain why some CSS quirks are the way they are.

Other Resources

The textbook for the CMWEB 120 course serves more than as a one-time read as I learn more about HTML and CSS. This book includes charts and tables that lay out various information that I frequently need to go back to, as having all of the basic properties memorized is impractical.

Whenever I have trouble recalling how I did a certain thing CSS or HTML previously, I go back to one of my earlier labs, especially those from CMWEB 110. Doing so helps me remember exactly how I implemented some of the more fundamental aspects of CSS because many of those labs are simpler in nature and often would be the first time I used that specific element or property.
