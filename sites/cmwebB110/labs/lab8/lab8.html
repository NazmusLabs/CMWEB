So, Why Use Floats?

Floats are part of a larger shift in web development that was taking place in full shift by the mid-2000s. Float works in conjunction with the div tag. You see, before CSS, the only way to organize your webpage in any other format other than a sequence of headers and paragraphs one after another was to use tables. Initially, tables met the web layout needs of the time, but as websites became more complex, the need for a more flexable approach to laying out a web page was becoming clear. That is where float comes in.

The Div Element

We can use opening and closing "div" tags to create a div element. The div element allows us to divide (as the tag name suggests) our web page into sections. And on their own, these divisions will follow one after another, from top to bottom, as is the case for all elements of an un-styled HTML document. However, when combined with CSS and its float property, we begin to see the power and flexibility of using CSS to lay out a web page.

P.S. With CSS handling the web layout, we now use tables for what they were originally intended to do: create standard tables to present data.

The Width Property

By default, the content of an HTML element (such as those inside the div tags) will span to cover the entire width of its container, from top to bottom and left to right. (I am defining container as the parent HTML element or, if no parent tags exist, the browser window itself.)

With CSS, however, we can change this. By setting a percentage value of the width and height style properties of an HTML element, we can have the content of the tag span only the given percent of the container’s width and height. So, for example, if we set the width value of an HTML element to be 80%, it would mean that 80% of the width of the container would be filled with the content (text, picture, etc.) while the remaining 20% of the width would be empty space. By default, the content of the HTML element is centered, with 10% of the empty space going to the left of the content and the 10% of the empty space going to the right of the content. But what if we want to have the content be left-aligned, with the entire 20% of the empty space being on right of it? To do this, we use the float property.

The Float Property

Using float, we can tell the content of an HTML element to be left-aligned, center-aligned, or right-aligned. We can set the width property of a div element to be 50%, for instance, and set the float property to be left-aligned. This would make it such that the left half of the container is full of the content while the right half of the container is empty space. Now, here’s where it gets interesting. We can now float the following HTML element, which, by default, displays below our current div element. By the setting element to have a width of 50% and its float property set to left-aligned, we can place this next element to the right of our div element.

Important: We must remember reset the layout at the end of the floating elements; otherwise, it can really mess up the layout of the following element(s). This can be done using the "clear" property with the value set to "both". I create a class called ".clearfix" that includes the clear property set to "both". After all the floating elements, we need to create one more div element with no content in between the tags and set the class for that div element to the ".clearfix" aforementioned ".clearfix" class. This essentially ensures all following HTML elements will return to the standard sequential layout. We must do this after each time we float elements.
